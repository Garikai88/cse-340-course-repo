export {getAllOrganizations, getOrganizationDetails};

const getOrganizationDetails = async (organizationId) => {
    const query = `
        SELECT 
            organization_id,
            name,
            description,
            contact_emai,
            logo_filename
        FROM organization
        WHERE organization_id = $1;
    `;

    const queryParams = [organizationId];
    const result = await db.query(query, queryParams);

    // We then return the first row of the result set, or null if no rows are found
    return result. rows.length > 0 ? result.rows[0] : null;
};

[Coach]: Thanks for sharing this snippet! Your model structure is really clean, and I love how you handle the query parameters.

Looking at your `getOrganizationDetails` function, you used a placeholder (`$1`) and passed `[organizationId]` in a separate `queryParams` array to your database execution method.

Can you explain why passing the parameter this way is a preferred security practice compared to just combining strings directly (like using template literals to inject the variable straight into the SQL text)?

-------------------------------------------------------------------

[Student]: This protects the database from malicious input which may distort the database.

-------------------------------------------------------------------

[Coach]: Spot on! That is exactly what it does. Using `$1` tells the database driver to treat that parameter strictly as literal data, completely neutralizing any malicious SQL code someone might try to sneak into the inputs (known as SQL Injection).
