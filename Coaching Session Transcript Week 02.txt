// src/models/categories.js
import db from './db.js'; 

export async function getAllCategories() {
    const queryText = `
    SELECT category_id, name
    FROM category
    ORDER BY name ASC;
    `;
    try {
        const result = await db.query(queryText);
        return result.rows;
    } catch (error) {
        console.error('Error executing getAllCategories query:', error);
        throw error;
    }
}

[Coach]: That is a beautifully clean and structured model function, Garikai! I notice that you used the `async` and `await` keywords here alongside a `try/catch` block. Can you explain why we need to use `await` when calling `db.query(queryText)`, and what would happen to our application if the database server suddenly went offline and we didn't have that `try/catch` block handling the error?

[Student]: This because most database operations are asynchronous . This means that the database will recieve a request from the application and then wait for the server to respond.

[Coach]: Exactly, you are right on the money with it being asynchronous! To clarify just a tiny bit on the direction of that wait: our Node.js application sends the query out to the PostgreSQL server, and then Node.js has to pause and wait for PostgreSQL to finish processing and send the rows back. Using `await` ensures our code doesn't jump ahead to the next line before that data actually arrives. And if the database goes offline without that `try/catch` block, an unhandled error would instantly crash our whole Node server, taking the entire website down! Your code prevents that beautifully.
